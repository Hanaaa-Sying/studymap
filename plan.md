# StudyMap 产品规划存档

---

## 2026-05-19 · MVP 阶段初始规划

### 核心决策
- **目标用户**：大学生，初始场景为博克尼大学政治学/社会学类期末考试
- **产品形态**：先做 CLI 脚本验证核心逻辑，后期演进为对外发布的产品
- **幕布集成**：生成幕布兼容的大纲 Markdown 格式，用户手动导入（不用非官方 API，避免稳定性风险）
- **AI Provider**：默认 DeepSeek（国内可用），通过 LLMClient 抽象层支持多 provider（豆包、Claude）

### 核心功能
1. **填充模式**：PDF/大纲 → AI 生成层级知识框架 → 导出幕布格式 → 用户逐节课填充
2. **复习模式**：论述题输入 → 定位相关知识节点 → 推荐高频话术 → 答题思路

### 知识框架粒度原则
框架太粗没用，太细在幕布里很乱。约束：最多 3-4 层，节点标题 ≤ 10 字，目标 50-80 节点/课。

### 遗留问题
- 幕布大纲导入格式需验证（缩进 Markdown vs 幕布专属格式）
- DeepSeek 在框架提取任务上的质量需实测
- 话术跨课程去重逻辑待设计
