## P01 — 知识框架生成

**用途**：从课程 PDF 或大纲文本中提取层级知识框架，输出结构化节点树。

**约束说明**：
- 最多 3-4 层层级（根 → 大主题 → 子主题 → 概念）
- 节点标题 ≤ 10 字，名词短语，不写完整句子
- 优先保留：理论名、学者名、核心命题、对比关系
- 过滤掉：章节编号、引言/结论性文字、举例性内容
- 目标节点数：50-80 个（适合一门完整课程）

**模板**：
```
你是一个专业的学术课程知识结构分析师。
以下是课程《{course_name}》的材料（大纲/教材节选）：

<material>
{material_text}
</material>

请提取该课程的知识框架，严格遵守以下规则：
1. 输出 JSON 格式的节点树，结构见下方 schema
2. 层级最多 4 层，不超过
3. 每个节点 title 不超过 10 个字，使用名词短语
4. 只保留考试相关内容：理论名称、学者姓名、核心命题、重要对比
5. 删除：章节编号、"引言"/"结论"/"举例"等非考点内容
6. 整个树的节点总数控制在 50-80 个

输出 JSON schema：
{
  "course": "{course_name}",
  "nodes": [
    {
      "id": "n1",
      "title": "...",
      "parent_id": null,
      "level": 1
    }
  ]
}

只输出 JSON，不要任何解释文字。
```

**更新历史**：
- 2026-05-19：初始版本
- 2026-05-22：规则 7（语言）由硬编码"不做翻译"改为 `{lang_instruction}` 占位符，由 `ai/lang_utils.build_lang_instruction()` 动态注入，支持原文/翻译/双语三种模式

## P02 — 高频话术提取

**用途**：从课程材料中识别可在论述题中复用的高频表达、学术话语。

**约束说明**：
- 提取的话术要能在论述题中"直接使用或稍作改动后使用"
- 每条话术附带适用场景描述
- 识别跨课程可复用的通用学术话语

**模板**：
```
你是一个社会科学论述题答题专家。
以下是课程《{course_name}》的材料：

<material>
{material_text}
</material>

请从中提取可在论述题中直接使用的高频话术和学术表达。

要求：
1. 每条话术是一个完整表达（1-3 句话），可直接嵌入答题
2. 附上适用场景（如"批判某理论时"、"阐述某概念的局限性时"）
3. 标注来源（页码或章节）
4. 优先提取：理论批判套路、概念界定句式、学者观点引用格式

输出 JSON 格式：
[
  {
    "text": "...",
    "context": "适用场景描述",
    "source": "页码/章节"
  }
]

只输出 JSON，不要解释。
```

**更新历史**：
- 2026-05-19：初始版本
- 2026-05-22：规则 5（语言）同步改为 `{lang_instruction}` 占位符，逻辑与 P01 一致

## P03 — 复习模式：知识定位 + 话术推荐

**用途**：给定论述题，定位相关知识节点并推荐相关话术。

**模板**：
```
你是一个考试答题助手。
学生面对的论述题是："{question}"

以下是该课程的知识框架节点（JSON）：
{framework_json}

以下是话术库（JSON）：
{rhetoric_json}

请：
1. 从框架中识别与该题目最相关的节点（列出 id 和 title，3-6 个）
2. 从话术库中推荐最适合答题的条目（2-4 条）
3. 给出一个 3 步答题思路（每步一句话）

输出格式：
{
  "relevant_nodes": [{"id": "...", "title": "..."}],
  "recommended_rhetoric": [{"text": "...", "context": "..."}],
  "answer_outline": ["第一步...", "第二步...", "第三步..."]
}
```

**更新历史**：
- 2026-05-19：初始版本
